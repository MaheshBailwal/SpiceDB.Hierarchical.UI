• Q. How to connect to Authorization/Spice DB.
    A. To connect to authorization DB you need to put SpiceDB url and SpiceDB token in respective input boxes and click on load button.
    Note: If you don't have access to SpiceDB running on Azure, then you have to set up SpiceDB on local machine, for that you can follow jira ticket https://wencosupport.atlassian.net/browse/CPF-1357
  • Q. How to perform action..
    A. For performing any action right click on tree node, where context menu will open which will show the action that can be performed against that selected item.
  • Q. How to Test.
    A. For testing click on Test button, test window will appear. On test window you can input user and organization for which you want to test. After that you can click on respective button like Get Assets, Get Devices and you will see the items in Result section for which user has permission.
  • Q. How to edit permission/relationship.
    A. You can delete relationship by right clicking on respective tree node/item and delete it, then you can add back modified/desired relationship again.
  • Q. How to import.
    A. You can import any valid yaml file that has data according to Wenco Authorization schema.
  • Q. How to export.
    A. You can export data by clicking on export button. Once export button is clicked check box will be available aganist all items in Authorization tree. You need to select items you want to export and after that click on Complete Export button. MessageBox will show up displaying the name of yaml file with exported data.
  • Q. What is use of export/import.
    A. Export/Import feature can be useful when we want to move data from one environment to another environment. Like deveplover can create authorization data on its local machine/docker and then export data that can be pushed to other cloud environments.
